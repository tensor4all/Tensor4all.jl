# Feature Test Report: Tensor4all.jl Contribution Workflow (Non-AI Contributor)

**Date:** 2026-04-16
**Project type:** Julia library (FFI wrapper)
**Features tested:** Contribution workflow from non-AI contributor perspective
**Profile:** ephemeral
**Use Case:** Expert Julia developer without AI tools wants to report a bug
**Expected Outcome:** CONTRIBUTING.md and issue templates work standalone without AI assumptions
**Verdict:** fail
**Critical Issues:** 1

## Summary

| Feature | Discoverable | Setup | Works | Expected Outcome Met | Doc Quality |
|---------|-------------|-------|-------|---------------------|-------------|
| Issue templates | yes | yes | yes | partial | missing version field |
| CONTRIBUTING.md | yes | yes | partial | no | AI-centric framing |
| PR workflow | partial | no | partial | no | no PR template |

## Per-Feature Details

### Issue templates (bug report)
- **Role:** Expert Julia developer, no AI tools
- **Discoverability:** Good. README → CONTRIBUTING.md → templates.
- **Functionality:** Template fields are well-structured.
- **Friction:** No Tensor4all.jl version or tensor4all-rs pin field. No "Actual behavior" field separate from Description.

### CONTRIBUTING.md
- **Role:** Same
- **Discoverability:** Good.
- **Functionality:** Core 4-step flow works for non-AI contributors.
- **Friction:** ~30-40% of content is AI-specific (skills section, AI tool instructions). Step 3 "Post detailed plan" is gated with "If you are using AI tools" — implies planning is only for AI users.

### PR workflow
- **Friction:** No PR template. PR checklist exists only in AGENTS.md (lines 268-270), not in CONTRIBUTING.md. Non-AI contributors would miss it.

## Issues Found

### Critical
1. **Non-AI contributors feel like second-class participants.** The contribution flow, planning step, and a significant portion of CONTRIBUTING.md frame AI usage as the primary path. Architecture information is in `AGENTS.md` — a name and framing that signals "for bots."

### Medium
2. **Step 3 gated on AI usage.** "If you are using AI tools, post an implementation plan" should be reframed as universally valuable.
3. **No PR template.** `.github/PULL_REQUEST_TEMPLATE.md` is missing.
4. **PR checklist only in AGENTS.md.** Should be duplicated in CONTRIBUTING.md.
5. **Bug template missing version fields.** No Tensor4all.jl version or tensor4all-rs pin.

### Minor
6. **CLAUDE.md contains only `@AGENTS.md`** — meaningless to human readers browsing the repo.
7. **No "Actual behavior" field** in bug report template.

## Suggestions

1. **Reframe Step 3** — remove "If you are using AI tools". Write: "For non-trivial changes, consider posting an implementation plan as a comment."
2. **Move AI skills section** to a subsection at the bottom of CONTRIBUTING.md or a separate file. Keep the main guide focused on the universal flow.
3. **Add PR template** (`.github/PULL_REQUEST_TEMPLATE.md`).
4. **Copy PR checklist** from AGENTS.md into CONTRIBUTING.md.
5. **Add version fields** to bug report template (Tensor4all.jl version, tensor4all-rs pin).
6. **Add "Actual behavior" field** to bug report template.
